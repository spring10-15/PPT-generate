# Fixed Template PPT

## 目标

把用户上传的 `doc/docx` 素材整理成：

- 封面：标题 / 副标题 / 署名 / 日期
- 目录页：固定 3 栏目录
- 详情页：按模板参考页的固定槽位填充

导出的 `.pptx` 必须直接复用模板页，不能自己重画版式。

## 核心原则

1. 以模板 XML 为准，不以目测样式为准。
2. 目录页直接使用模板最后一页。
3. 详情页直接复用模板中的参考页。
4. 只替换文本和图片关系，不改版式、坐标、字体、颜色、行距。
5. 单页内容放不下时优先拆页，不把超长原文硬塞进一个文本框。

## SOP 对应关系

- 节点一：文档解析与大纲生成
- 节点二：Python 模板扫描与版式特征提取
- 节点三：大纲与版式的智能匹配
- 节点四：数据灌入与文件生成

## 模板拆解流程

1. 读取模板 `.pptx`。
2. 用 Python 扫描脚本解包 `ppt/slides/slide*.xml`、对应 `.rels` 以及 `slideLayout/slideMaster` 继承链。
3. 识别固定参考页：
   - `slide1`：封面
   - `slide22`：目录
   - `slide4/5/9/10/11/12/13/15/16`：详情页参考版式
4. 对每个文本坑位提取 shape name、占位符类型、物理位置、字号和安全字符数；若 slide 本身缺失这些参数，则从 `slideLayout` / `slideMaster` 回填。
5. 在 shape name 精确命中之外，再用示例文本、字号、位置和占位符类型做一层半自动角色识别，辅助识别 `pageTitle / intro / regularTitle / smallTitle / detailBody`。
6. 通过 shape name 填充固定槽位，例如 `Title 1`、`Rectangle 18`、`TextBox 20`。
7. 导出时重新生成 `presentation.xml`、`presentation.xml.rels` 和 `[Content_Types].xml` 的 slide 清单。

## 目录页规则

目录页固定使用 `slide22`，不再新绘制目录页。

固定槽位：

- `Title 1`：固定写 `目录`
- `TextBox 11` x 3：目录标题
- `TextBox 12` x 3：目录简要说明
- `Rectangle 2 / 18 / 22`：模板自带 `一 / 二 / 三`，不改

目录内容规则：

- 目录项最多 3 项
- 单个目录标题不超过 5 个字
- 单个目录简要说明不超过 10 个字
- 目录标题和简述只填纯文本，不额外添加 `【一】/【二】/【三】`
- 不展示“目录总览 / 目录分组 / 页码范围 / 共几项 / 共几页”

## 素材结构化规则

先按文档原始顺序抽取章节，再做轻度聚合：

- 目录项按顺序生成，尽量不超过 3 项
- 一个目录项可以跨多页
- 多个目录项不能合并到同一页
- 只允许补少量连接词，不允许引入新事实

每页详情只保留以下结构：

- `参考版式`：页标题
- `内容简介`
- `常规标题`
- `小标题`
- `说明性文字`

`强调标题` 已删除，不再作为结构化字段输出。

## 字数与拆页规则

固定控字规则：

- `参考版式` 不超过 10 个字
- `内容简介` 不超过 30 个字
- `常规标题` 不超过 15 个字
- 单个 `常规标题` 下最多 3 个 `小标题`
- 单个 `小标题` 不超过 15 个字
- 单个 `小标题` 下的 `说明性文字` 不超过 60 个字
- 限字按单块内容生效，不是按整页总字数统一裁短

实现原则：

- 节点二扫描出的 `maxChars` 作为物理上限来源
- 业务红线与物理上限同时存在时，取更严格者
- 标题框和正文框若在 slide 层没有几何信息，必须沿 `slideLayout / slideMaster` 继承链补齐后再估算承载量

拆页规则：

1. 先对正文做句级切分，优先按 `。；，、` 拆。
2. 若单句仍过长，再按长度二次切分。
3. 单页槽位装不下时，把同一目录项内容继续拆到下一页。
4. 已经占用的内容不重复回填到同页其他槽位。
5. 不足以填满所有槽位时，剩余槽位留空，不造假文案。

## 去重规则

同一页内以下内容尽量不重复：

- 页标题
- 内容简介
- 常规标题
- 小标题
- 说明性文字

实现方式：

- 先做标准化：去空格、去编号、去常见标点
- 再按顺序去重
- 常规标题优先保留；小标题和说明性文字跳过重复片段

## 页型选择规则

先看内容密度，再决定页型：

- `overview`：概述型、内容较少
- `two-column`：左右分栏且带图
- `three-column`：中量内容，三栏均衡
- `four-column`：高密度短句内容
- `progress`：步骤、推进、计划、路径类内容
- `vertical`：上下结构
- `split-grid`：上下两块、每块多要点
- `hierarchy`：架构、体系、分层类内容
- `table`：素材存在表格时优先
- `image / image-left / image-right`：素材存在原图时优先

同一种页型允许重复使用。

## 详情页回填规则

所有详情页都遵循两条：

1. `Title 1` 填页标题
2. `文本框 39` 填内容简介

文字样式固定覆写：

- `参考版式`：28pt，`#0070C0`
- `内容简介`：16pt，`#0070C0`
- `常规标题`：14pt，`#EFEFEF`
- `小标题`：14pt，`#000000`
- `说明性文字`：16pt，灰色主题值

其余槽位按版式填充：

- `slide4`：左右结构 1:1，带图；第二个旧“强调标题”槽位留空
- `slide5`：左右结构 1:1:1
- `slide9`：左右结构 1:1:1:1
- `slide10`：推进图
- `slide11`：上下结构
- `slide12`：上下分栏密集结构
- `slide13`：概述页
- `slide15`：层级页
- `slide16`：表格页

处理要求：

- 优先占满真实槽位
- 不复用同一段文字填多个槽位
- 内容少时优先选择简单版式，不为了多样性强行切复杂页型
- 旧模板示例词必须在导出前清空

## 图片规则

- 只使用用户文档原图
- 没有原图时保留模板图片占位
- 图片页仍使用模板参考页，只替换关系文件中的图片目标

## 导出规则

1. 克隆模板 slide XML 与 `.rels`
2. 按页型选择参考页
3. 回填 shape 文本
4. 有图时替换图片关系目标
5. 清理模板残留占位词
6. 重写演示文稿 slide 清单

## 更新模板时的检查清单

当模板文件更新时，先做这几步：

1. 统计总页数是否变化
2. 确认封面页是否仍是 `slide1`
3. 确认目录页是否仍是最后一页
4. 重新列出目录页和详情页的 shape name
5. 检查是否新增、删除或重命名了重复 shape
6. 回归导出后检查是否还残留模板示例词
